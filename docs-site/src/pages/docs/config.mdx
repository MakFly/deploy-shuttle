`.deployshuttle.yml` lives at the project root. The CLI walks parent
directories until it finds one. Override with `--config <path>`.

## Full schema

```yaml
version: 1

app:
  domain: app.example.com         # Used by TLS, HSTS, DNS, health-endpoint checks.
  healthcheckPath: /health        # GET https://<domain><path> must return 2xx.

checks:
  profile: [docker, caddy]        # Default: docker, caddy.
  ignore:
    - adminer.ip_restriction_missing
    - ssh.port_default

docker:
  allowDockerSocket:
    - dozzle*                     # Glob: prefix*, *suffix, exact.
    - portainer_agent*
  allowRoot:
    - shared_redis
  workerServices:
    - "*-queue"
    - "*-horizon"
    - "*-scheduler"

cloudflare:
  enabled: true
  zone: example.com
  tokenEnv: CLOUDFLARE_API_TOKEN  # Override env var name if needed.
```

Every key is optional. An empty file is valid (`version: 1` is implied).

## `app`

| Key | Type | What it does |
| --- | --- | --- |
| `domain` | string | Public hostname. Unlocks `tls.cert_missing`, `tls.hsts_missing`, `dns.domain_not_pointing_to_server`, `monitoring.no_health_endpoint`. |
| `healthcheckPath` | string | Path probed for `monitoring.no_health_endpoint`. Skipped without it. |

## `checks`

| Key | Type | What it does |
| --- | --- | --- |
| `profile` | string list | Categories to load. Default: `docker, caddy`. |
| `ignore` | string list | Check IDs to mark as skipped (no score impact). |

Ignored findings still appear in reports under "Accepted risks" so client
reviewers can see them.

## `docker`

| Key | Type | What it does |
| --- | --- | --- |
| `allowDockerSocket` | string list | Workloads that legitimately mount `/var/run/docker.sock` (Dozzle, Portainer agent, autoheal). |
| `allowRoot` | string list | Workloads that must run as root. |
| `workerServices` | string list | Background workers without an HTTP healthcheck (Laravel queue, Sidekiq, etc.). |

All three lists support exact names plus simple prefix (`prefix*`) and
suffix (`*suffix`) wildcards.

## `cloudflare`

| Key | Type | What it does |
| --- | --- | --- |
| `enabled` | boolean | Master switch. Cloudflare checks skip when `false`. |
| `zone` | string | Zone name (e.g. `example.com`). |
| `tokenEnv` | string | Env var holding the API token. Defaults to `CLOUDFLARE_API_TOKEN`. |

Token must have read scope on `Zone`, `DNS`, `Zone Settings`. The token
itself is **never** written to disk.

## Generated by `init --preset`

The `init --preset` command writes a curated `.deployshuttle.yml` per
stack. See [`init`](/docs/commands/init) for the four available presets.
