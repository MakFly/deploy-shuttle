DeployShuttle is a CLI-first **production-readiness** tool for Docker apps
on VPS. It connects to a server (locally or over SSH), runs **43 checks**
across 12 categories, computes a deterministic 0-100 score, and either
prints a console report or generates a Markdown / HTML / PDF report you
can hand to a client.

It does **not** replace your deploy tooling. It tells you whether the box
underneath is safe enough to host the deploy.

## Who this is for

- Freelance and agency developers shipping client apps on Hetzner / OVH /
  Contabo / Scaleway VPS.
- Indie hackers running side-projects on cheap arm64 boxes.
- Teams that want a CI gate that fails the build when the target server
  is below a readiness threshold.

## Three commands you will actually use

```bash
deploy-shuttle doctor --target root@server
deploy-shuttle harden --target root@server --dry-run
deploy-shuttle report --format html --output report.html
```

Each one is documented in the [Commands](/docs/commands) section.

## What gets checked

A full inventory lives in the [Check catalog](/docs/checks). Highlights:

- Public database ports, UFW state, Docker port mappings.
- SSH hardening (root login, password auth, port 22 default).
- TLS certificate validity and HSTS header.
- Cloudflare SSL/TLS mode, WAF, proxy state, DNS records.
- Compose file hygiene (`:latest` tag, missing env files, sensitive bind
  mounts, missing resource limits).
- Backup detection (artifacts + cron jobs).

## Free vs Pro

The free CLI covers everything you need to scan and ship. **Pro** unlocks
the hosted HTML / PDF reports you hand to clients and the remote
`harden --apply` workflow. See [Pricing](/pricing).
