Shuttle is a CLI-first tool that takes you from code to production on any
VPS. It detects your stack, generates production Dockerfiles, provisions a
server with Docker Swarm and Caddy, deploys with zero-downtime rolling
updates, and runs **43 checks** across 12 categories to give you a
deterministic 0-100 readiness score.

One CLI. Init, provision, deploy, audit, harden.

## Who this is for

- Freelance and agency developers shipping client apps on Hetzner / OVH /
  Contabo / Scaleway VPS.
- Indie hackers running side-projects on cheap arm64 boxes.
- Teams that want zero-downtime deploys without Kubernetes overhead.
- Anyone who needs a CI gate that fails the build when the target server
  is below a readiness threshold.

## Three commands to production

```bash
shuttle init
shuttle provision
shuttle deploy
```

## Audit and harden

```bash
shuttle doctor --target root@server
shuttle harden --target root@server --dry-run
shuttle report --format html --output report.html
```

Each command is documented in the [Commands](/docs/commands) section.

## What gets checked

A full inventory lives in the [Check catalog](/docs/checks). Highlights:

- Public database ports, UFW state, Docker port mappings.
- SSH hardening (root login, password auth, port 22 default).
- TLS certificate validity and HSTS header.
- Cloudflare SSL/TLS mode, WAF, proxy state, DNS records.
- Compose file hygiene (`:latest` tag, missing env files, sensitive bind
  mounts, missing resource limits).
- Backup detection (artifacts + cron jobs).

## What gets deployed

- 3 strategies: Swarm (rolling updates), Compose, Blue-Green.
- FrankenPHP Dockerfiles for Laravel and Symfony.
- Docker Secrets encrypted at rest, RAM-only in containers.
- Auto-TLS via Caddy + Let's Encrypt.
- Rollback support for all strategies.
