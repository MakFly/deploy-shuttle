The install script detects your OS / architecture, downloads the matching
binary from the latest GitHub release, verifies the SHA-256 checksum, and
drops the binary into `~/.local/bin`.

## One-liner (Linux and macOS)

```bash
curl -fsSL https://raw.githubusercontent.com/MakFly/deploy-shuttle/main/scripts/install.sh | sh
```

The script prints what it is doing and warns you if `~/.local/bin` is not
on your `$PATH`.

## Pre-built targets

| OS / arch | Asset name |
| --- | --- |
| Linux x86-64 | `shuttle-linux-x64` |
| Linux ARM64 (Hetzner CAX, AWS Graviton, Pi 4/5) | `shuttle-linux-arm64` |
| macOS Intel | `shuttle-darwin-x64` |
| macOS Apple Silicon | `shuttle-darwin-arm64` |

For other targets, build from source:

```bash
git clone https://github.com/MakFly/deploy-shuttle
cd deploy-shuttle/go-cli
go install ./cmd/shuttle
```

## Pin a specific version

```bash
SHUTTLE_VERSION=v2.0.0 \
  curl -fsSL https://raw.githubusercontent.com/MakFly/deploy-shuttle/main/scripts/install.sh | sh
```

## Custom install directory

```bash
SHUTTLE_INSTALL_DIR=/usr/local/bin \
  curl -fsSL https://raw.githubusercontent.com/MakFly/deploy-shuttle/main/scripts/install.sh | sudo sh
```

## Verify the install

```bash
shuttle --version
# shuttle version v2.0.0

shuttle doctor --help
```

## Auto-update

Shuttle checks for new versions on startup (non-blocking, 3s timeout).
When a newer release is available, it prints a one-line notice. To update:

```bash
shuttle update
```

## Uninstall

```bash
shuttle uninstall
```

This removes the binary and the `~/.shuttle/` directory. If you prefer
manual removal:

```bash
rm "$(command -v shuttle)"
rm -rf ~/.shuttle
```
