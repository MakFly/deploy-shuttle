The install script detects your OS / architecture, downloads the matching
binary from the latest GitHub release, verifies the SHA-256 checksum, and
drops the binary into `~/.local/bin`.

## One-liner (Linux & macOS)

```bash
curl -fsSL https://raw.githubusercontent.com/MakFly/deploy-shuttle/main/scripts/install.sh | sh
```

The script prints what it is doing and warns you if `~/.local/bin` is not
on your `$PATH`.

## Pre-built targets

| OS / arch | Asset name |
| --- | --- |
| Linux x86-64 | `deploy-shuttle-linux-x64` |
| Linux ARM64 (Hetzner CAX, AWS Graviton, Pi 4/5) | `deploy-shuttle-linux-arm64` |
| macOS Intel | `deploy-shuttle-darwin-x64` |
| macOS Apple Silicon | `deploy-shuttle-darwin-arm64` |

For other targets, build from source:

```bash
git clone https://github.com/MakFly/deploy-shuttle
cd deploy-shuttle/go-cli
go install ./cmd/deploy-shuttle
```

## Pin a specific version

```bash
DEPLOY_SHUTTLE_VERSION=v1.0.1 \
  curl -fsSL https://raw.githubusercontent.com/MakFly/deploy-shuttle/main/scripts/install.sh | sh
```

## Custom install directory

```bash
DEPLOY_SHUTTLE_INSTALL_DIR=/usr/local/bin \
  curl -fsSL https://raw.githubusercontent.com/MakFly/deploy-shuttle/main/scripts/install.sh | sudo sh
```

## Verify the install

```bash
deploy-shuttle --version
# deploy-shuttle version v1.0.1

deploy-shuttle doctor --help
```

## Uninstall

```bash
rm "$(command -v deploy-shuttle)"
```

The CLI does not write outside `~/.local/bin` by default, except for the
optional license file at `~/.deployshuttle/license.json` and the latest
report cache at `.deployshuttle/latest-report.json` inside your project.
