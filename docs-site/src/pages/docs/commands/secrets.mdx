`secrets` manages application secrets with a two-layer model: secrets are
encrypted locally with Argon2id + XChaCha20-Poly1305, then pushed to the
Docker Swarm cluster as Docker Secrets (encrypted at rest in the Raft log,
mounted as RAM-only files inside containers).

## Usage

```bash
shuttle secrets <subcommand> [flags]
```

## Subcommands

### set

```bash
shuttle secrets set <KEY> <VALUE>
```

Stores a secret in the local encrypted store at `.shuttle/secrets.enc`.
The passphrase is read from the `SHUTTLE_SECRETS_PASSPHRASE` environment
variable or prompted interactively.

```bash
shuttle secrets set APP_KEY "base64:xxxxxxxxxxxxx"
shuttle secrets set DB_PASSWORD "s3cret"
shuttle secrets set REDIS_PASSWORD "r3d1s"
```

### get

```bash
shuttle secrets get <KEY>
```

Reads a single secret from the local store. Requires the passphrase.

### list

```bash
shuttle secrets list
```

Lists all secret keys in the local store (values are not printed).

### remove

```bash
shuttle secrets remove <KEY>
```

Removes a secret from the local store.

### push

```bash
shuttle secrets push [--target <user@host[:port]>]
```

Pushes all local secrets to the remote server as Docker Secrets. On a
Swarm cluster, secrets are stored encrypted in the Raft log and mounted
into containers at `/run/secrets/<key>` -- never written to disk on the
host.

## Docker Secrets (Swarm strategy)

When using the Swarm deploy strategy, `shuttle secrets push` creates
Docker Secrets that your services reference in the compose file. The
lifecycle:

1. **Local**: secrets are encrypted with your passphrase in
   `.shuttle/secrets.enc`.
2. **Push**: `shuttle secrets push` creates or updates Docker Secrets on
   the Swarm manager.
3. **Deploy**: `shuttle deploy` references the secrets in the stack
   definition.
4. **Runtime**: containers read secrets from `/run/secrets/<key>` (tmpfs,
   RAM-only).

## Env file split (Compose / Blue-Green)

For non-Swarm strategies, secrets are managed via a file split:

- `.env` -- configuration values, safe to commit.
- `.env.secrets` -- secret values, `chmod 600`, never committed.

The `doctor` check suite verifies `.env` file permissions and Git
tracking automatically.

## CI / non-interactive usage

In CI pipelines or scripts, set the passphrase via environment variable:

```bash
export SHUTTLE_SECRETS_PASSPHRASE="your-passphrase"
shuttle secrets set DB_PASSWORD "$DB_PASSWORD"
shuttle secrets push
```

## Examples

```bash
# Add secrets for a Laravel app.
shuttle secrets set APP_KEY "base64:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
shuttle secrets set DB_PASSWORD "production-password"
shuttle secrets set MAIL_PASSWORD "smtp-password"

# List what is stored.
shuttle secrets list

# Push to production server.
shuttle secrets push --target deploy@app.example.com

# Remove a secret.
shuttle secrets remove MAIL_PASSWORD
```

## Security model

- Local encryption: Argon2id key derivation + XChaCha20-Poly1305
  authenticated encryption.
- Passphrase never stored on disk.
- Docker Secrets: encrypted in Swarm Raft log, decrypted only in
  container memory (tmpfs).
- The `doctor` check `secrets.env_world_readable` verifies that `.env`
  files are not world-readable on the server.
