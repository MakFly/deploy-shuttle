Shuttle exposes commands for two workflows: **deploy** (init, provision,
deploy, rollback, secrets) and **audit** (doctor, harden, report). Every
command is documented in this section.

| Command | What it does |
| --- | --- |
| [`init`](/docs/commands/init) | Detect your stack, generate Dockerfile + docker-compose.yml + config. |
| [`provision`](/docs/commands/provision) | Bootstrap a bare VPS with Docker Swarm + Caddy + UFW. |
| [`deploy`](/docs/commands/deploy) | Build and deploy using Swarm, Compose, or Blue-Green strategy. |
| [`rollback`](/docs/commands/rollback) | Rollback to the previous deployment. |
| [`secrets`](/docs/commands/secrets) | Manage encrypted secrets locally and push as Docker Secrets. |
| [`doctor`](/docs/commands/doctor) | Run the 43 readiness checks locally or over SSH. |
| [`harden`](/docs/commands/harden) | Build a remediation plan from the latest doctor report. |
| [`report`](/docs/commands/report) | Render the latest doctor report as Markdown, HTML, or PDF. |
| [`license`](/docs/commands/license) | Activate, refresh, or check the status of your Pro license. |

## Other commands

```text
shuttle status      Show container status across servers
shuttle logs        Stream remote logs
shuttle ssh         Open SSH session to server
shuttle exec        Run command in remote container
shuttle monitor     Live Docker resource usage
shuttle badge       Generate score badge for README
shuttle update      Self-update to latest version
shuttle uninstall   Remove shuttle from this machine
```

Run `shuttle <command> --help` for usage details.

## Global flags

Two flags are accepted by most commands:

- `--format console|json` -- output shape. `json` is what CI / scripts
  consume.
- `--config <path>` -- point at an explicit `.shuttle.yml`. By default
  the closest one in parent directories wins.

## Exit codes

| Code | Meaning |
| --- | --- |
| `0` | Success / readiness threshold met. |
| `1` | Threshold failure or critical finding. |
| `2` | Connection or configuration error. |
| `3` | Internal error (please file an issue). |

Use these in CI gates rather than parsing stdout.
