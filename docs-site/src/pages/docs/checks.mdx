Every finding has a stable `id`, a severity, a short summary, and a
remediation hint. The list below is the contract for downstream tooling
that reads doctor JSON output.

## Severity & scoring

| Severity | Deduction | Behavior |
| --- | --- | --- |
| `critical` | 20 | Causes `doctor` to exit non-zero regardless of `--fail-below`. |
| `high` | 10 | Counted toward score; counts as an open finding. |
| `medium` | 5 | Counted toward score. |
| `low` | 2 | Counted toward score. |
| `info` | 0 | Surfaced in reports, no score impact. |

## System

| ID | Severity | What it verifies |
| --- | --- | --- |
| `system.os_supported` | high | Host runs Ubuntu 22.04, Ubuntu 24.04, or Debian 12. |
| `system.disk_space_low` | medium / high | Root filesystem free space (high above 90%). |
| `system.updates_pending` | medium / high | Pending APT upgrades (high above 50). |
| `system.memory_low` | medium / high | Available memory ≥ 10% of total. |
| `system.unattended_upgrades_inactive` | medium | unattended-upgrades package installed and enabled. |
| `system.fail2ban_inactive` | medium | fail2ban active. |
| `system.swap_missing` | low | At least one swap device or file. |
| `system.time_sync_inactive` | medium | A time-sync daemon is active. |

## Cloudflare

Cloudflare checks are opt-in. Set `cloudflare.enabled: true`, your zone,
and an API token (env var `CLOUDFLARE_API_TOKEN` by default) in
`.shuttle.yml`. Without these, every Cloudflare check skips with a
clear explanation.

| ID | Severity | What it verifies |
| --- | --- | --- |
| `cloudflare.ssl_flexible` | critical | SSL/TLS mode is not Off or Flexible. |
| `cloudflare.always_https_disabled` | medium | "Always Use HTTPS" enabled. |
| `cloudflare.waf_disabled` | medium / info | Managed WAF enabled (skipped on Free plan). |
| `cloudflare.dns_missing` | high | A / AAAA / CNAME record exists for `app.domain`. |
| `cloudflare.proxy_disabled` | medium | Host records proxied (orange cloud). |
